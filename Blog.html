html forms in php and working with files ...✎

| march 5th 2023 | 11:59pm ☾

The main differences between $_GET and $_POST are visibility and size limitations. With $_GET, information sent from a form, such as variable names and values, are visible to everyone in the browser's address bar and has a size limit of 100 characters for any value. With $_POST, information sent from a form is not visible to others and the size limit is 8 mb, but this limit can be changed within the php.ini file. Typically $_GET is good to implement if a website may want to be bookmarked, but $_POST is better when dealing with potentially sensitive information within forms, such as passwords.
